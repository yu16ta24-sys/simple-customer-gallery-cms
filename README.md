# Simple Customer Gallery CMS - Stable Phase 4

## 内容
- ログインURL分離
- customer_manager ロール
- 専用CMSダッシュボード
- ギャラリーカテゴリ
- 写真アップロード
- 写真管理
- ドラッグ並び替え
- 説明文編集
- 論理削除

## 確認
1. プラグイン停止・削除
2. このZIPをアップロード
3. 有効化
4. 専用CMS → 写真管理
5. カテゴリ選択
6. 画像一覧、編集、削除、並び替えを確認


## 0.5.0
- 写真管理の編集UIを、画像カード内ではなくクリック行の下に横幅いっぱいで差し込む方式へ変更。


## 0.5.1
- 編集エリア内のサムネイルを削除。
- 選択画像に対応する吹き出しポインターを追加。
- 削除ボタン文言を「画像を削除する」に変更。
- 確定後に編集エリアを自動で閉じるよう変更。


## 0.6.0
- 写真管理に「通常 / 削除済み」切替を追加。
- 削除済み画像の復元機能を追加。
- 管理者のみ完全削除機能を追加。


## 0.6.1
- 写真管理のカテゴリ選択をプルダウンからボタン式UIに変更。
- 親カテゴリ選択に応じて子カテゴリボタンを自動生成。


## 0.7.0
- 写真管理UIをApple系ミニマルに調整。
- カテゴリ未選択時の案内ボックスを追加。
- 通常/削除済みタブを廃止し、単一ボタンの復元モード切替に変更。


## 0.7.1
- 削除済み画像復元モードを赤文字・右寄せに変更。
- ドラッグ並び替え時の見た目をふわっとしたアニメーションに調整。


## 0.7.2
- 写真管理の通知を右下トースト表示に変更。
- 成功通知は緑、エラー通知は赤。
- ふわっと表示して自動で消える動作を追加。


## 0.7.3
- 通知と復元モードボタンをカード右下に配置。
- 通知を緑色のピル型に調整。
- ホバーの微妙な色変化、クリック時scale(.98)、並び替えの慣性・浮遊感を強化。


## 0.9.1
- 写真追加をギャラリー管理へ統合。
- 下部ボタンを左右同じ高さに配置。
- 管理画面内アップロードを実際のアップロード処理に接続。
- アップロード後に一覧を自動更新。


## 0.9.2
- ギャラリー管理内アップロードに進捗バーを追加。
- XMLHttpRequestでアップロード進行率を表示。


## 0.9.3
- アップロード100%後の一覧更新待ち時間を短縮。
- アップロード完了後すぐに一覧を再読み込み。


## 0.9.4
- 進捗バーの100%表示タイミングを修正。
- 送信完了時は95%で止め、WordPress側の画像処理完了後に100%表示。
- 画像処理中の表示を追加。


## 0.9.5
- ダッシュボードのギャラリー管理ボタン重複を修正。
- 死んでいる写真追加リンクを削除。


## 0.9.6
- アップロード最大枚数を5枚から10枚に変更。
- アップロード前プレビューを1列化し、説明文入力欄を横長に拡張。


## 0.9.7
- アップロード待機画像を追加方式に変更。
- 各待機画像に個別キャンセルボタンを追加。
- 新しく画像を選んでも既存の待機画像と説明文を保持。


## 0.9.8
- お客様アカウントでも全画像を表示する仕様に変更。
- アップロード者による表示制限を撤廃。
- 完全削除のみ管理者限定を維持。
