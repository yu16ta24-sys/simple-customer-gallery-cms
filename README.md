## データ構造仕様

### scg_photo

ギャラリー画像は、WordPressのカスタム投稿タイプ `scg_photo` として1画像につき1投稿で管理する。

meta:

- `_scg_attachment_id` 実画像のattachment_id
- `_scg_order` 同一サブカテゴリ内での表示順。型はint想定
- `_scg_status` `active` / `hidden`
- `_scg_uploaded_by` アップロードしたユーザーID

taxonomy:

- `scg_gallery_category`
  - 親カテゴリ = メインカテゴリー
  - 子カテゴリ = サブカテゴリー

並び順:

- 同一サブカテゴリ内で `_scg_order` 昇順

---

### scg_blog / scg_news

ブログとお知らせは、それぞれ独立したCPTで管理する。

```txt
scg_blog = ブログ
scg_news = お知らせ
```

基本データ:

```txt
post_title   = タイトル
post_content = 本文
post_status  = publish / draft
post_author  = 投稿者
```

meta:

```txt
_scg_status = active / hidden
_scg_image_1 = attachment_id
_scg_image_2 = attachment_id
_scg_image_3 = attachment_id
```

削除:

- 通常削除は `_scg_status = hidden`
- 復元時は `_scg_status = active`
- 完全削除は管理者のみ
- 完全削除時は添付画像も削除する

## 設計上の判断基準

このプラグインは、コード上はWordPressの投稿・メディア・タクソノミーを利用しているが、設計上は「お客様用の専用CMS」である。

そのため、判断基準は以下。

```txt
WordPressとして正しいか
よりも
お客様が迷わず操作できるか
```

優先順位:

```txt
1. 誤操作しにくい
2. 画面が分かりやすい
3. 動作が気持ちいい
4. WordPress標準機能を壊さない
5. 将来拡張しやすい
```
