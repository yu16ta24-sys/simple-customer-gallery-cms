# Simple Customer Gallery CMS - v1.0.2

## 現在のバージョンとフェーズ名

- Version: 1.0.2
- Phase: v1.0.2 整理・安全化版
- 位置づけ: v1.1.0 のブログ/お知らせ管理に進む前の安定化フェーズ

---

## 今回実装した内容

- プラグイン本体のバージョンを `1.0.2` に更新
- 画像アップロード時の1ファイル上限を20MBに設定
- 20MBを超える画像を選択した場合、管理画面側とサーバー側の両方でエラー表示
- エラー文言を「ファイルサイズが大きすぎます。20MB以内の画像を選択してください。」に統一
- 旧「写真追加」単独ページ用の残骸を整理
  - `scg-photo-upload` の許可ページ指定を削除
  - `admin-upload.js` の読み込み処理を削除
  - 未使用の `admin-upload.js` を削除
  - 未使用の `render_upload_page()` を削除
- ギャラリー取得・並び順取得の `WP_Query` に `meta_type => NUMERIC` を追加
- `_scg_order` の数値ソートをより明確化
- README内の表記を「写真管理」ではなく「ギャラリー管理」に統一
- READMEをGeminiレビュー用の引き継ぎ書として更新

---

## 【Geminiへのレビュー依頼事項】

今回の v1.0.2 は大きな新機能追加ではなく、ブログ/お知らせ管理へ進む前の整理・安全化フェーズです。

重点的に見てほしい点：

1. `class-scg-gallery-upload.php` の20MB制限の実装が安全か
   - 複数枚アップロード時の挙動
   - WordPress/PHP側のアップロードエラー処理
   - `media_handle_upload()` 前の検証順序

2. 旧「写真追加」単独ページの残骸整理で、必要な処理まで消していないか
   - ギャラリー管理内アップロードは `admin-manage.js` 側で継続
   - Ajaxアクション `scg_upload_photos` は継続使用

3. `_scg_order` のソート補強として `meta_type => NUMERIC` の追加位置が適切か

4. Apple風UI/UXとして、20MB超過時のエラー通知が自然か
   - 現状は既存トースト通知 `#scg-manage-message` に表示
   - もっと分かりやすい導線があれば提案希望

5. v1.1.0 で予定しているブログ/お知らせ管理に進む前に、今のギャラリー管理まわりで追加整理すべき点がないか

## 動作確認手順

1. WordPress管理画面で旧バージョンのプラグインを停止・削除
2. このZIPをアップロード
3. プラグインを有効化
4. 専用CMS → ギャラリー管理
5. メインカテゴリーとサブカテゴリーを選択
6. 画像一覧、説明文編集、削除、復元、並び替えを確認
7. 20MB以内の画像をアップロードできるか確認
8. 20MBを超える画像でエラー表示されるか確認
9. customer_manager アカウントで通常管理画面に入れないことを確認
10. 管理者のみ完全削除ボタンが表示されることを確認
